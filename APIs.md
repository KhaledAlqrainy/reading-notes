# APIS

### What does REST stand for?

* Representational State Transfer State Transfer (REST) 

### REST APIs are designed around ?

* Resource 

### What is an identifer of a resource? Give an example.

* URI that uniquely identifies that resource.

### What are the most common HTTP verbs?

* GET, POST, PUT, PATCH, and DELETE.

### What should the URIs be based on?

* Nouns not verbs.

### Give an example of a good URI.

* https://adventure-works.com/orders

### What does it mean to have a ‘chatty’ web API? Is this a good or a bad thing?

* “chatty” web APIs that expose a large number of small resources, and its a bad thing that we should avoid it.

### What status code does a successful GET request return?

* HTTP status code 200 (OK).

### What status code does an unsuccessful GET request return?

* 404 (Not Found).

### What status code does a successful POST request return?

* code 200.

### What status code does a successful DELETE request return?

* HTTP status code 204 (No Content)
